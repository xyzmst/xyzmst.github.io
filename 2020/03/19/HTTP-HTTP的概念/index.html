Q:125.001.HTTP.001
请求报文由哪些组成
A:
请求行： method+路径+http版本
Header
body

Q:125.001.HTTP.002
常用的method
A：

get

  • 用于获取数据
  • 对服务器数据不进行修改
  • 不发送body

post

  • 用于增加或修改资源
  • 发送给服务器的内容写在body里面

put

  • 用于修改数据
  • 发送给服务器的内存写在body里面

delete

  • 用于删除资源
  • 不发送body

head

  • 和get使用方法完全相同
  • 和get唯一区别在于返回的响应中没有body
    put get （幂等性） 操作一次和多次 一样

Q:125.001.HTTP.003
响应报文组成
A:
状态行 版本+状态码+message
header
body

Q:125.001.HTTP.004
常用状态码
A:
1xx: 临时消息 如：100 （继续发送）、101（正在切换协议）
2xx:成功 200（OK）、201（创建成功）
3xx：重定向 如 301（永久移动）、302（暂时移动）、304（内容未改变）
4xx: 客户端错误 如 400（客户端请求错误）、401（认证失败）、403（被禁止）、404（找 不到内容）
5xx:服务端错误 500（服务器内部错误）

Q:125.001.HTTP.005
常用的头信息

A:

host

  • 目标主机
  • 注意：不是在网络上⽤于寻址的，而是在目标服务器上用于定位子服务器的

content_type

  • 指定 Body 的类型 主要有四类

content_length

  • 指定 Body 的长度（字节）

transfer:chunk 分段

  • (分块传输编码 Chunked Transfer Encoding)
  • 用于当响应发起时，内容长度还没能确定的情况下。和 Content-Length 不同时使⽤用。用途是尽早给 出响应，减少用户等待。

location

  • 指定重定向的目标 URL

User-Agent

range/accept-range

  • 按范围取数据
    • Accept-Range: bytes 响应报⽂文中出现，表示服务器器⽀支持按字节来取范围数据
    • Range: bytes=- 请求报⽂文中出现，表示要取哪段数据
    • Content-Range:-/total 响应报⽂文中出现，表示发送的是哪段数据
  • 作⽤：断点续传、多线程下载。

其他

Accept: 客户端能接受的数据类型。如 text/html
Accept-Charset: 客户端接受的字符集。如 utf-8
Accept-Encoding: 客户端接受的压缩编码类型。如 gzip Content-Encoding：压缩类型。如 gzip

Q:125.001.HTTP.006
content—type 有哪4类
A:
text/html

  • 请求 Web 页面是返回响应的类型，Body 中返回 html ⽂本

x-www-form-urlencoded

  • 网页纯文本提交方式

multitype/form-data
- web页面含有二进制文件时的提交方式

application/json , image/jpeg , application/zip …
- 单项内容（⽂文本或⾮非⽂文本都可以），⽤用于 Web Api 的响应或者 POST / PUT 的请求

Q:125.001.HTTP.007
cache作⽤
A:
在客户端或中间⽹络节点缓存数据，降低从服务器取数据的频率，以提高网络性能。
